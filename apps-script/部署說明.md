# 佑發空調 案件成本利潤管理系統 — 部署說明

## 這是什麼
一個 Google Apps Script 網頁應用，用來管理獨立施工案件的**成本、報價、利潤**，並可產生報價單 PDF。
資料庫是程式在你 Google Drive 裡自動建立的一份試算表（名稱「佑發空調_案件資料庫」）。
只要開同一個網址，兩台 Windows 就是看同一份資料、即時同步。

檔案：
- `Code.gs` — 後端程式（資料存取、報價單 PDF、PDF 辨識）
- `Index.html` — 前端畫面
- 本檔 — 部署步驟

---

## 一次性部署（約 5 分鐘）

1. 開 <https://script.google.com> → **新專案**
2. 把 `Code.gs` 內容貼進預設的 `Code.gs`（全選覆蓋）
3. 左側「＋」→ **HTML** → 命名為 **Index**（一定要叫 Index）→ 貼入 `Index.html` 內容
4. **啟用 Drive 進階服務**（PDF 讀取功能需要）：
   左側「服務」→「＋」→ 找到 **Drive API** → 新增
5. 右上「部署」→「新增部署」→ 類型選「網頁應用程式」
   - 執行身分：**我自己**
   - 誰可以存取：**只有我自己**（若老闆要自己開，改成「知道連結的任何人」或指定網域）
6. 第一次會要求授權存取 Drive／試算表 → 允許 → 完成後會得到一個網址
7. 把網址加到兩台電腦的瀏覽器書籤

> 之後若你改了 `Code.gs` 或 `Index.html`，要「部署 → 管理部署 → 編輯（鉛筆）→ 版本選『新版本』→ 部署」才會更新到同一個網址。

## 使用

- **首頁**：新增案件、看所有案件的成本／報價／利潤／純利潤率（紅黃綠分級）
- 點**案號**進明細：填報價項目、記成本、下載 PDF 報價單
- 報價可從「常用項目」帶入，或用**歷史單價查詢**帶入舊價
- **預留成本率**：發獎金前先扣一段預留（可選以「成本」或「報價淨額」為基準），
  純利潤 = 帳面利潤 −（基準 × %），這才是真正可分配的利潤

### PDF 報價單自動讀取
在案件明細頁「報價項目」上方，可上傳舊報價單 PDF 自動辨識：
- **電子檔 PDF（電腦匯出、佑發格式）辨識最準**；掃描／拍照的也能讀但需多校對。
- 上傳 → 讀取 → 出現「讀取結果」表 → 核對修正品名／單位／數量／單價。
- 按「帶進本案報價」放進目前案件（記得再按**儲存報價**），或「存進歷史庫」供日後查詢帶入，或兩個都按。
- 辨識靠 Google Drive OCR，暫存的辨識檔用完會自動刪除。

## 想調整的地方（都在 `Code.gs` 最上面的設定區）

| 要改什麼 | 改哪裡 |
| --- | --- |
| 營業稅率 | `TAX_RATE`（預設 0.05 = 5%） |
| 報價單抬頭、承辦人、電話、信箱、有效天數 | `COMPANY` 物件 |
| 常用項目預設清單 | 直接編輯試算表「常用項目」工作表，或改 `ensureDatabase` 內的預設 |

---

## 老闆看得到什麼
設定成他能存取後，他開網址就能看到所有案件的成本、報價、利潤、純利潤率，
以及每個案子可下載的 PDF 報價單。目前是可讀可操作；若要限制成唯讀，之後可再加權限。

## 這一版做了哪些優化（相對於初版）

- **修正致命錯誤**：初版 `makeQuotePdf` 缺了函式宣告，整個 `Code.gs` 無法載入、任何功能都跑不動 → 已補回，PDF 報價單可正常產生。
- **兩台同時操作更安全**：所有寫入（新增案件／成本、儲存報價、刪除、匯入歷史）都加上 `LockService` 鎖，避免兩台電腦同時存檔互相覆蓋。
- **擋重複案號**：新增案件時若案號已存在會擋下並提示，不再產生重複資料。
- **儲存報價／刪除改整批重寫**，取代逐列刪除，速度更快也更不易出錯。
- **總覽彙總改用雜湊表**，案件變多時載入更快（不再逐案掃描全部成本與報價）。
- **前端加上失敗提示**：後端出錯（如重複案號、網路中斷）會跳出提示，不再靜默失敗；按鈕在處理中會鎖住避免連點。
- **報價單抬頭改用 `COMPANY` 設定**，不用再進函式內部改字。
